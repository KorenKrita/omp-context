🧠 Council Verdict: Compact Benefit Formula

omp-context · 2026-06-27 · 5 advisors + peer review
问题：设计 omp-context 的 compact 收益指标公式，给 agent 作为定量参考（非自动决策）。要求利用可用数据、可解释、反映"零成本/无缓存/可逆"机制、与 bash-agent 收益性指标相反（鼓励更激进 compact）。

最终公式

Benefit(a) = Δtokens(a) × pressure(p) × decay(c) Δtokens(a) = Σ entry∈[a,HEAD] (getMsgContent(entry).length+3)/4 + 1 pressure(p) = p / (100 - p) // p = getContextUsage().percent decay(c) = 0.9 ^ c // c = branch 上 branch_summary 数
  • Δtokens × pressure（主收益）：腾出的空间 × 当前压力。压力低分数自然低，压力高飙升。
  • decay(c)（唯一温和反向项）：历史 compact 次数越多收益折扣越大。防 summary 叠加漂移 + 回查负担。c=5 仍剩 59%，不违背"鼓励激进"。
  • 无 S / D / recoverability：公理或不可观测，移出公式。reversibility 在 SKILL 文字说明。
  • 排序分非阈值分：agent 对比候选 anchor 选最高，不设"超过 X 才 compact"。

1. Council 收敛点 （高置信）

  • 草稿三致命硬伤：S 极性反转（分母抑制 compact）、D 决策时刻无值可取（summary 是手写传入的）、recoverability 是公理非变量。
  • 真问题是排序非阈值：compact 近免费+可逆，"值不值得"默认 yes，agent 需要的是"选哪个 anchor"的排序分。
  • 公式应极简：bash-agent 5 项成本在 omp-context ②③④ 全归零，复杂公式是"用复杂度伪装精确性"。

2. Council 分歧 （真实张力）

  • 要不要公式：First Principles/Outsider 倾向极简 H×P；Executor/Contrarian 要 3 项带 Span。裁决：3 项但主指标呈现 2 项。
  • pressure 形态：p²/(100-p) 爆炸 vs p/(100-p) 温和。裁决：用 p/(100-p)，鼓励激进但不失控。

3. Peer Review 揭露的盲点 （最有价值）

  • per-entry token 估算代码库不存在（reviewer 4 实测）：getMsgContent 只有截断逻辑，contextUsage.tokens 是窗口级总量。任何公式需先补 token 估算 helper。
  • labels 和 compact count 被全员闲置（reviewer 2/3/5）：compact count 可作温和谨慎系数。
  • 回查成本是 reversibility 公理下唯一真实反作用力（reviewer 3）：agent 回头展开 off-path 要付 token，用 compact count 代理。

5. 第一件事

先把公式写进 SKILL.md 作为 agent 参考框架（零代码风险），验证 agent 能否用好"Benefit = 腾出空间 × 压力 × 衰减折扣"心智模型，再决定是否做成 acm_timeline HUD 自动计算。

理由：council 揭露最大风险是 per-entry token 估算函数不存在，代码实现有实际工作量。心智模型可先通过 SKILL 文字传递，零成本验证 agent 是否买账。若 agent 用文字指引就够，连代码都不用写。

Contrarian 完整响应
Contrarian草稿三硬伤：S 在分母极性反转；D 不可计算（无 summary quality 信号，自指）；H 当"丢失收益"前提崩塌（可逆=moved not lost）。P(p)=p²/(100-p) 爆炸。真实成本（切回摩擦、summary 纠错）失明。承认数据不足退回 2-3 维，比凑五变量公式负责。
First Principles 完整响应
First Principlesomp-context 真需要公式吗？手动+手写已交还判断权。S 不可量化、D≈0 爆炸、recoverability 常数零区分。②③④ 归零只剩"agent 写 summary 认知成本"不可量化。只有 H 和 P 可量化。Score=H×P。"opposite polarity"是改决策默认值非公式符号。范式错配。
Expansionist 完整响应
Expansionist草稿低估自己。S 应压倒性大项。真问题"哪个 anchor 释放未来余量最大"。H→H×density。最大 upside：context OS 雏形，有效 context window 趋无限。数据缺口用历史均值估。
Outsider 完整响应
Outsider三卡点：问错问题（免费+可逆"值得"默认 yes，该排序非阈值）；recoverability 放分母反向（该删，是常量）；三量两看不见（summary_weakness 无来源，distance≈dropped_size 打架）。score=freed×pressure 两项。可逆性是公理不是变量。
Executor 完整响应
ExecutorCompactBenefit(a)=Δtokens×p/(100-p)/(Span+1)。Δtokens=anchor→HEAD token 和；p=percent；Span=entry 数。opposite polarity 靠 p→100 飙升 + 积极倾向，不引入 S。Monday：acm_timeline HUD 加 computeCompactBenefit，O(n)，acm_compact details 复用。
