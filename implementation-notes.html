ACM Implementation Notes

Current implementation decisions, updated 2026-07-12. Superseded cross-repository synchronization decisions were removed when omp-context became independently maintained.

Decision Record

ProvenanceDecisionRationale and constraint
user-decided Maintain omp-context as an independent repository. The repository no longer owns or synchronizes another product repository. Product documentation, verification, release state, and host compatibility are local concerns only.
user-decided Track the exact locally installed OMP host through a pre-commit contract gate. Every commit detects the local coding-agent, agent-core, and pi-ai versions. A changed version is validated in an isolated cold candidate before package metadata, the adapter constant, and locks are updated. Failed candidates leave the repository version untouched.
user-decided Remain extension-only and synchronize live AgentSession state through one pinned adapter. branchWithSummary() remains the persistent tree mutation. After a definitively successful travel, the exact-version adapter rebuilds the active branch and calls agent.replaceMessages() at the matching tool_execution_end.
user-decided Isolate OMP internals behind one narrow Host Bridge. All guarded SessionManager mutations live at this seam. Domain logic and agent-facing tools consume typed structural results rather than casting host internals directly.
user-decided Verify runtime behavior with the real SessionManager and AgentSession. The isolated fixture uses the exact declared OMP packages without a model or API key and proves resolver origin, mutation behavior, context reconstruction, compaction lifecycle, live replacement, repeated travel, resume, and session isolation.
user-decided The ACM Skill package is the sole guidance authority. Canonical CORE owns the normal path. Generated system-prompt text, tool descriptions, cues, and recovery fragments are derived artifacts rather than editable behavioral sources.
user-decided Retain one model-invoked advanced Skill and the fixed seven-slot handoff. Advanced guidance loads only for ambiguous target selection, interleaved fronts, archive round trips, or exceptional recovery. Every handoff contains Goal, State, Evidence, External, Exclusions, Recover, and NEXT.
user-decided Use hard structural invariants and soft semantic criteria. Runtime validates observable shape and mutation state. Boundary quality, cold start, recoverability sufficiency, and whether NEXT is genuinely executable remain agent completion criteria.
user-decided Use one strict timeline view discriminator. The public views are active, checkpoints, search, and tree. Legacy booleans, unknown keys, blank queries, and view-incompatible fields are rejected.
agent-resolved Model host mutations as explicit transaction outcomes. Mutation ports return not_applied, applied, or indeterminate after observing the real journal and leaf. Compensation uses an operation-scoped token; applied or indeterminate branch changes create a context-refresh obligation.
agent-resolved Use a version-checked, identity-scoped live AgentSession adapter. The adapter installs one idempotent wrapper at AgentSession.getContextUsage(), associates SessionManager → WeakRef<AgentSession> by object identity, and schedules replacement only for the matching successful travel tool call.
agent-resolved Preserve persistent recovery when live synchronization is unavailable or fails. Unsupported host shape, missing association, stale work, build failure, or replacement failure never rolls back an otherwise valid travel. The summary branch and backup checkpoint remain authoritative, and public context rebuild continues.
user-decided Add semantic rebase as a normal-path agent transition, not a new tool. A rebase carries all surviving state in one authoritative snapshot to the earliest safe base, with root as the ideal candidate rather than a default. acm_travel remains the only mutation primitive.
user-decided Use cold start as the hard rebase gate. A fresh agent must be able to execute NEXT from the snapshot and direct evidence pointers without reading archived summaries. Context pressure triggers the check but never lowers it.
agent-resolved Expose summary-depth evidence without runtime safety verdicts. Timeline counts active branch_summary nodes separately from native compaction, exposes root as a structural candidate, and projects post-travel depth. Travel reports before/after/delta facts but never labels a candidate safe.
user-decided Refine rebase without weakening checkpoint, local-fold, failed-direction, or task-end behavior. The always-on CORE retains every normal transition and a complete local-fold example. Rebase adds a structural-reset criterion alongside cold start rather than replacing the existing fold gate.
agent-resolved Use one advanced reference at a time and reroute when the observable condition changes. This preserves progressive disclosure without trapping archive recovery inside the wrong playbook after a failure. Recovery references now distinguish backup-label rollback from indeterminate branch mutation and use factual refresh/live-sync lifecycle states.

Open Decisions

None. A failed local-host cold candidate or a new official OMP navigation API must reopen the pinned adapter decision.
