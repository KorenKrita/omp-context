ACM Implementation Notes

Created 2026-07-11 before implementation; updated 2026-07-12 for semantic rebase. These decisions were resolved during ACM grilling and specification sessions.

Decision Record

ProvenanceDecisionRationale and constraint
user-decided omp-context is the canonical ACM repository. magic-acm-context consumes ACM through a one-way manual sync. The integrated repository is not an independent implementation authority.
user-decided Synchronization remains manual. Expand the local sync command to cover implementation, guidance, generated artifacts, and tests. Do not add CI parity, webhooks, scheduled synchronization, or automatic pull requests.
user-decided Support exactly OMP 16.4.5. @oh-my-pi/pi-coding-agent, @oh-my-pi/pi-agent-core, and @oh-my-pi/pi-ai must share the exact version. This replaces the previous exact 16.4.2 contract after changelog, imported-surface, type, and isolated real-host verification.
user-decided Remain extension-only and synchronize live AgentSession state through one pinned adapter. branchWithSummary() remains the persistent tree mutation. After a definitively successful travel, the exact OMP 16.4.5 adapter rebuilds the active branch and calls the associated AgentSession's agent.replaceMessages() at the matching tool_execution_end. No OMP source patch or command-only navigation API is required.
user-decided Isolate OMP internals behind one narrow Host Bridge. All guarded SessionManager capabilities live at this seam. Domain logic and agent-facing tools consume typed structural results rather than casting host internals directly.
user-decided Verify runtime behavior with the real SessionManager. Use an isolated OMP 16.4.5 fixture without a model or API key. Pin resolver roots and prove all OMP imports come from the fixture.
agent-resolved OMP 16.4.5 requires no ACM compatibility shim. The 16.4.2 → 16.4.5 audit found unchanged SessionManager, session-entry, tokenizer, compaction, and PI AI type surfaces used by ACM. Session-context only added an opt-in transcript option; the breaking task-tool wire schema is outside ACM's extension contract.
user-decided The ACM Skill package is the sole guidance authority. A canonical CORE supplies the normal path. Generated system-prompt and tool guidance are derived artifacts, not editable behavioral sources.
agent-resolved Use one canonical CORE producer and one prompt owner per deployed extension. Standalone ACM registers its public before_agent_start hook. The integrated consumer disables that hook and calls ensureAcmCoreSegment() from its sole prompt orchestrator, preserving unrelated segment boundaries.
user-decided Use one model-invoked advanced Skill. CORE owns the normal path. The Skill triggers only non-obvious target selection, interleaved fronts, missing anchors, archive round trips, and exceptional recovery.
user-decided Keep three representative examples always on. Burst, failed direction, and finished task chain provide transferable usage patterns. Advanced reference is split by cognitive task rather than by one Skill per scenario.
user-decided Retain the fixed seven-slot handoff. Goal, State, Evidence, External, Exclusions, Recover, and NEXT always appear; empty categories use none. This is an agent completion criterion, not runtime proof of semantic correctness.
user-decided Use hard structural invariants and soft semantic heuristics. The plugin validates observable structure only. Boundary quality, fold value, raw recoverability sufficiency, and whether NEXT is genuinely executable remain agent judgments.
user-decided Context pressure triggers boundary inspection. Pressure does not independently authorize travel. Native token compaction remains OMP's responsibility and the plugin never cancels, replaces, or delays it; travel remains a semantic-boundary operation.
user-decided Simplify timeline to one strict view discriminator. Use active, checkpoints, search, or tree. Reject legacy booleans, unknown keys, whitespace-only search/filter values, and fields invalid for the selected view.
user-decided Do not add a travel preview/apply protocol. Timeline provides candidate evidence. Travel performs side-effect-free prevalidation and then applies one recoverable transition.
user-decided Tool results use progressive disclosure and raw evidence. Success returns concise facts and one next cue; exceptional paths return relevant recovery guidance. Remove threshold-based effect labels and report raw before/after/delta and structural message facts.
user-decided README is maintainer-facing. Document installation, architecture, exact version, sync direction, verification, and known limits. Link to canonical agent guidance instead of duplicating it.
user-decided Validate guidance through static contracts and real use. Do not create a stochastic model benchmark now. Change guidance in response to observed failures or host-contract changes.
agent-resolved Model host mutations as an explicit transaction outcome. Mutation ports return not_applied, applied, or indeterminate after observing the real label journal and leaf. Travel compensation uses an operation-scoped token; applied or indeterminate branch changes always create a context-refresh obligation.
agent-resolved Publish the synchronized ACM surface transactionally. Transforms assert source match cardinality, staged outputs receive independent structural validation, and a rollback journal restores every replaced destination if publication or final verification fails. Per-file temporary rename alone was insufficient because it allowed a partial consumer state.
agent-resolved Keep one OMP synchronization authority in the consumer. The scheduled workflow invokes canonical sync-acm.mjs plus --verify-only. The legacy OMP copier and injector were removed. A checked consumer-owned patch restores only the integration seam after upstream Magic Context sync and fails closed on drift.
agent-resolved Bind consumer verification to derived provenance. verify:acm is non-mutating. Exact host versions derive from package metadata, and acm-provenance.json records canonical/host versions, manifest hash, and every synchronized artifact hash.
agent-resolved Split runtime behavior by ownership rather than file size. src/index.ts is a short composition root. Checkpoint, timeline, travel, transaction coordination, lifecycle, message sanitation, entry resolution, and label replay each have one owning module.
agent-resolved Use a version-checked, identity-scoped live AgentSession adapter. src/live-agent-session-adapter.ts supports exactly OMP 16.4.5. It installs one idempotent wrapper at AgentSession.getContextUsage(), associates SessionManager → WeakRef<AgentSession> by object identity, and exposes only structured synchronization outcomes. Host Bridge remains the sole owner of label and branch mutations.
agent-resolved Apply live replacement only after the matching successful travel tool execution. Only a definitively applied branch mutation schedules work for its resulting leaf. The matching tool_execution_end rebuilds and sanitizes messages from that leaf, preserving the host-owned in-flight tool call/result pair while making the next provider context and native stored-context accounting reflect the traveled branch. Repeated travel is leaf-checked so obsolete work cannot overwrite a newer branch.
agent-resolved Preserve persistent recovery when live synchronization is unavailable or fails. Unsupported host version/shape, missing association, stale work, message-build failure, or replacement failure never rolls back an otherwise valid travel. The branch summary and backup checkpoint remain authoritative; public context rebuild continues, diagnostics expose the exact state, and terminal failures include reload guidance.
agent-resolved Treat the private lifecycle seam as a replaceable compatibility boundary. The pinned-host matrix verifies successful shrinking travel, complete in-flight tool pairing, native accounting, repeated travel, off-path restoration, persistence/resume, failure recovery, lifecycle cleanup, and multi-session/subagent isolation. Future OMP upgrades must revalidate this seam. If OMP exposes an official AgentSession refresh or tree-navigation API to extensions, replace and remove the private adapter rather than maintaining both paths.
user-decided Add semantic rebase as a normal-path agent transition, not a new tool. A rebase carries all surviving state in one authoritative snapshot to the earliest safe base, with root as the ideal candidate rather than a default. The existing acm_travel mutation remains the only execution primitive; no acm_rebase tool or decorative travel mode is added.
user-decided Use cold start as the hard rebase gate. A fresh agent must be able to execute NEXT from the snapshot and direct evidence pointers without reading archived summaries. Context pressure triggers this check but never lowers it; local fold, live detail, or native compaction remain valid when the gate fails.
agent-resolved Expose summary-depth evidence without runtime safety verdicts. Timeline counts active branch_summary nodes separately from native compaction, exposes root as a structural candidate, and projects post-travel depth. Travel reports before/after/delta facts. Runtime never labels a candidate safe or claims that a completed mutation passed cold start.
